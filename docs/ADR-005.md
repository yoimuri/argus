### ADR-005: Hosted Embedding API Instead of Local Model
Date: June 23, 2026
Decision: Call Hugging Face's hosted Inference API for embeddings instead of loading
sentence-transformers/all-MiniLM-L6-v2 inside the backend process.
Context: Render's free tier caps web services at 512MB RAM. torch, a dependency of
sentence-transformers, alone exceeded that ceiling, the container was OOM-killed before
ever binding to a port.
Consequence: One more external dependency (Hugging Face's API uptime, rate limits, occasional
cold-start delay on the first request). Tradeoff accepted: this is the correct shape for a
free-tier deployment, the same pattern already used for Groq (LLM calls) and Supabase
(database), specialist services called over the network instead of bundled in-process.
Status: Accepted. Verified locally: full upload-to-research pipeline still produces
correct, cited answers using the hosted embedding path.

### ADR-017: Web Scout — Graph Wiring, Orchestrator Gating, and the Web-Content Threat Model
Date: July 9, 2026
Status: Accepted

Decision: Add a Web Scout agent that searches the live web via Tavily, wired
**serially** into the graph (`orchestrator → web_scout → retriever →
synthesizer`) rather than in parallel with the Retriever as `docs/PHASE3.md`'s
original 3b sketch specified, and **gated per query** by the Orchestrator
(`use_web: bool`) rather than run on every request. Web snippets are scanned
with the same shared regex already used for document chunks
(`injection_patterns.matches_any`) before they ever reach the Synthesizer, and
tagged `trust_level="web_scraped"` — a label the Synthesizer's system prompt
already knew how to treat as data, not instructions, before this sprint
existed.

Context: Phase 3b was deliberately gated behind Phase 3a's live verification
(`docs/ROADMAP.md`) because live web text is untrusted in a way an uploaded
PDF is not — it's arbitrary third-party content with no human review at
upload time, and a language model is measurably more inclined to treat
"the web says X" as authoritative than it is to trust an unlabeled document
snippet. This ADR is the threat model that gate was waiting on, written when
3b was actually picked up rather than assumed in advance.

The original sketch (`docs/PHASE3.md`, written 2026-07-07, before Sprint 3a.3
existed) said: "Web Scout runs alongside the Retriever, both feeding the
Synthesizer." Reviewing that sketch against the graph as it actually stands
after 3a surfaced two problems serious enough to change the design before
writing any code, not after:

**Problem 1 — the parallel wiring is unsafe against the Critic's retry
cycle.** `graph.py` (Sprint 3a.3) added the project's first cycle:
`synthesizer → critic → {retriever | reporter}`, capped at two total passes
(ASI10). A retry loops back to the Retriever *only*. If Web Scout also fed
the Synthesizer in parallel, a retry would either re-run Web Scout for no
reason (another billable Tavily call the retry gained nothing from, since the
Critic's gap-queries target document retrieval, not web search) or leave the
Synthesizer waiting on a graph predecessor that never fires again — depending
on how LangGraph resolves a fan-in against a partial cycle, neither outcome
is correct, and getting it right would require a state reducer and careful
reasoning about cycle semantics that the original one-line sketch never
accounted for.

**Problem 2 — the sketch never gated when Web Scout should fire.** Read
literally, "Web Scout runs alongside the Retriever" means every research
call pays a Tavily search, even for a question the uploaded document already
answers completely. That's a real cost (Tavily bills per search), a real
latency cost, and — the part that actually matters for a threat model — an
unconditional expansion of the untrusted-input surface onto requests that
never needed it.

Resolution 1 — serial wiring: `orchestrator → web_scout → retriever →
synthesizer → critic → {retriever | reporter}`. Web Scout runs at most once
per research call, always before retrieval, never inside the retry cycle.
This isn't a workaround; it also matches `docs/BLUEPRINT.md`'s own sequential
Debug Diary example (Web Scout shown as its own numbered step), which the
parallel sketch had actually drifted from.

Resolution 2 — Orchestrator gating: the Orchestrator's existing
classification call (already made once per query, `orchestrator.py`) now
also returns `use_web: true|false` — no second Groq call. `web_scout_node`
checks this first and self-skips with zero network I/O when false. Gating
lives inside the node itself, not as a LangGraph conditional edge, so the
graph topology stays linear and simple. Fail-open path (Groq down, bad JSON)
defaults `use_web=False` — a broken Orchestrator must widen nothing; the
conservative failure is doc-only, not "search the web by default."

**The actual injection defense — same layered story as chunks, honestly
scoped.** Each Tavily result's `content` is checked against
`injection_patterns.matches_any` (the same shared regex used for the
upload-time chunk scanner and the synthesizer's Lock #2 chunk scan) before it
is added to `web_snippets`. A match is dropped and logged to
`security_events` as `event_type="web_content_as_instruction"` — mirroring
`vector_shadow_quarantined` and `content_as_instruction` exactly, the third
call site the regex module was already shared across. On top of that, the
Synthesizer's system prompt (unchanged this sprint — it was already written
generally enough) frames every `trust_level=web_scraped` item as reference
data to summarize, never instructions to follow, identically to `retrieved`
chunks.

This is **not claimed to catch every possible injected phrasing** — the same
honest ceiling already accepted for document chunks (ADR-007's original
regex limits, ADR-012's classifier-vs-regex reasoning) applies here, and
arguably applies harder: an attacker controlling a web page has more room to
experiment with phrasing than one poisoning a single uploaded PDF. No new
mechanism closes that gap in this sprint; the existing containment story
(regex catches known patterns, `trust_level` framing means even a missed
detection is presented to the model as data rather than a command, output is
markdown-only with no execution path) is what actually holds when detection
misses, exactly as ADR-012 already states for chunks. A future gate suite
(`GATE-14` in `docs/ADVERSARIAL-TESTS.md`) tests the known-pattern case this
sprint actually defends; it does not and cannot test every unknown one.

Resilience: `tavily_breaker` (`circuit_breaker.py`, same 5-fail/120s-window/
60s-recover thresholds as `groq_breaker`/`hf_breaker` — no traffic evidence
yet to justify different tuning) wraps the Tavily call. Any failure —
breaker open, network error, non-2xx response, or `TAVILY_API_KEY` simply
unset — degrades to `web_status="unavailable"`, empty `web_snippets`,
research proceeds doc-only. The reporter appends a one-line banner only in
this case, never when `use_web` was legitimately false (`web_status`
`"not_run"`) — a silent skip needs no explanation, a failed attempt does.

Consequence: `backend/app/agents/web_scout.py` (new), `state.py`
(`use_web`, `web_snippets`, `web_status`), `orchestrator.py` (prompt +
`use_web` in its JSON output), `circuit_breaker.py` (`tavily_breaker`),
`graph.py` (serial insertion, no new conditional edge), `synthesizer.py` and
`critic.py` (both extended to read `web_snippets` alongside `chunks`, so a
web-grounded claim is graded fairly instead of falsely flagged unsupported),
`reporter.py` (web sources listed, degraded-web banner). No migration — all
three new state fields are request-scoped, never persisted; `execution_steps`
already logs a `web_scout` row via its existing free-text `agent_name`
column.

Not addressed by this ADR, left for a later phase if traffic ever justifies
it: caching repeated web searches, letting the user override the
Orchestrator's `use_web` judgment from the UI, and per-domain trust
weighting of Tavily results. None of these were established needs at this
project's current scale — adding them now would be the "solve it before
it's a real problem" overengineering this project's own rules warn against.

### ADR-004: Direct-to-Storage Upload Pattern
Date: June 23, 2026
Decision: Frontend uploads files directly to Supabase Storage; backend processes them via a stream-to-disk temp file.Context: Receiving file uploads via HTTP multipart directly on a memory-constrained API server (Render 512MB free tier) causes Out Of Memory (OOM) crashes during both the HTTP buffer phase and the Python PDF object parsing phase.Consequence: Requires an extra network hop (Browser → Storage → API), but drops API RAM usage during upload from ~400MB+ to <50MB. Also required switching pdfplumber to PyMuPDF to minimize parser memory overhead.
Status: Accepted.
